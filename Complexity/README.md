## 复杂度分析

- 时间复杂度：指执行当前算法所消耗的时间

- 空间复杂度：指执行当前算法需要占用的（临时）内存空间  
  *如果算法执行所需要的临时空间不随着某个变量n的大小而变化，即此算法空间复杂度为一个常量，可表示为 O(1)*

### 常见的复杂度

| 大O记发 | 非正式术语 |
|:---:|:---:|
| O(1)   | 常数阶    |
| O(logn)| 对数阶    |
| O(n)   | 线性阶    |
| O(nlogn)| 线性对数阶|
| O(n<sup>2</sup>)| 平方阶 |

### 时间复杂度

#### 1. 常数阶O(1)
无论代码执行了多少行，只要是没有循环等复杂结构，那这个代码的时间复杂度就都是O(1)，如：
```
int i = 1;      // 执行一次
int j = 2;      // 执行一次
int m = i + j;  // 执行一次
```

#### 2. 对数阶O(logn)
```
int i = 1;      // 执行一次
while(i<=n) {   // 执行logn次
    i = i * 2;  // 执行logn次
}
```
当i>n时，循环停止。也就是说少个2相乘后其结果值会大于n，即2<sup>x</sup>=n。  
那么x = log<sub>2</sub>n
